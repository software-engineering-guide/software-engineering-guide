# The documentation site

How the guidebook is built into a website and published. The site lives at
<https://collisdigital.github.io/software-engineering-guide/>.

## Toolchain

- **[Zensical](https://zensical.org/)** is the static site generator. It is
  configured in [`zensical.toml`](zensical.toml) and renders everything under
  `docs/` into the `site/` directory (which is not committed).
- **[uv](https://docs.astral.sh/uv/)** manages the Python toolchain and
  dependencies. They are declared in [`pyproject.toml`](pyproject.toml) and
  pinned in `uv.lock`.
- **[just](https://github.com/casey/just)** is the command runner. The recipes
  live in the [`justfile`](justfile).

## Building locally

Install [uv](https://docs.astral.sh/uv/getting-started/installation/) and
[just](https://github.com/casey/just#installation), then:

```sh
just           # list all tasks
just build     # build the site into site/
just serve     # serve the site locally with live reload
just test      # run the repository validation suite
just check     # regenerate navigation, then validate
```

`just build` runs `uv run zensical build --clean`, and uv installs the locked
dependencies into `.venv/` on first use, so no separate install step is needed.
Alternatively, open the repository in the [dev container](.devcontainer/devcontainer.json),
which comes with uv and just preinstalled.

## Content layout

Everything under `docs/` is published:

- `docs/index.md` : the home page (generated).
- `docs/front-matter/` : the opening essay, introduction, and contents page.
- `docs/chapters/` : the 100 chapters, named `N.M-slug.md`.
- `docs/spec/` : the specification (structure, conventions, roadmap).
- `docs/examples/` : worked examples.
- `docs/contributing/` : contributor guides and shared snippets.
- `docs/project/` : project documentation and the changelog.

## Navigation

The `nav` block in `zensical.toml` is generated by `tools/gen_nav.py` (run
`just nav`), together with the README table of contents, the site home page,
the contents page, and the subject index. The generator groups the chapters by
part, so the site navigation always mirrors the book's structure. The
validation suite (`just test`) fails if any page under `docs/` is missing from
the navigation.

## Cross-reference linking

Chapters refer to each other by decimal number in plain text ("see chapter
8.1"), per the house style. The `guide_xref` package at the repository root is
a Markdown extension, enabled in `zensical.toml`, that recognizes those
references at build time and links them to the matching chapter page. Its
chapter map (`guide_xref/chapters.py`) is generated by `just nav` and checked
by the validation suite; `tests/test_xref.py` tests the linking rules,
including the version numbers and quantities that must never be linked.

## Deployment

The GitHub Actions workflow
[`.github/workflows/docs.yml`](.github/workflows/docs.yml) runs on every push
to `main`. It installs the locked dependencies with uv, runs the validation
suite, builds the site with Zensical, and deploys the result to GitHub Pages
using the official Pages actions.

For the first deployment, GitHub Pages must be set to deploy from GitHub
Actions: repository **Settings → Pages → Build and deployment → Source →
GitHub Actions**.
